# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in any documentation, templates, or guidance that could lead to harm, please report it privately to [tito@marcolivi.it].

**Do not** open public issues for security concerns.

## What Constitutes a Security Issue

- Guidance that could enable manipulative AI behavior
- Templates that inadvertently normalize harmful practices
- Documentation gaps that could lead to ethical violations
- Framework elements that create dangerous dependencies

## Response Timeline

- Initial response: Within 48 hours
- Assessment: Within 1 week
- Fix implementation: Varies by severity
- Public disclosure: After fix, with your permission

## Scope

This security policy covers the ethical framework and templates. For security concerns about proprietary implementations, contact clients directly.
